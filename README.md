# Web Odds Archive

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面本身是完整、独立的 HTML 文档，不依赖于任何特定的服务器端环境或框架。本仓库仅作为这些页面的静态存档和发布点，不针对任何特定的域名、网站或服务。

## 目录结构

```
web-odds-archive/
├── pages/          # 存放所有 HTML 页面文件的目录
├── assets/         # 页面引用的公共资源（如图片、样式表、脚本）
├── index.html      # 仓库首页，提供页面导航或说明
└── README.md       # 本文件
```

- **pages/**：包含所有归档的独立 HTML 页面。每个页面以有意义的文件名命名，便于识别。
- **assets/**：集中存放页面可能引用的资源文件，如 CSS、JavaScript、图片等。各页面可通过相对路径引用。
- **index.html**：仓库的入口页面，通常包含页面列表、简要说明或使用指引。

## 页面归档说明

- 每个归档的 HTML 页面均为自包含（self-contained）文档，不依赖外部 API 或动态数据源。
- 页面内容、样式和逻辑均在单个 HTML 文件中定义，或通过相对路径引用本仓库 `assets/` 目录下的资源。
- 归档的页面可能来自不同来源、不同时间点，其内容与风格由原页面决定，本仓库仅负责保存与展示。
- 部分页面可能包含仅供存档参考的旧版内容或实验性功能。

## 使用方式

你可以通过以下方式查看归档页面：

1. 直接访问本仓库的 GitHub Pages（如果已启用）来浏览页面。
2. 将仓库克隆到本地，在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 若通过 HTTP 服务器访问，请确保服务器正确配置了静态文件服务。

## 维护说明

- 本仓库由维护者不定期更新，添加新的归档页面或更新现有页面。
- 提交新页面时，请将 HTML 文件放入 `pages/` 目录，并确保所有资源引用正确。
- 若页面引用了外部资源（如字体、库文件），建议下载后放入 `assets/` 目录，避免依赖失效。
- 欢迎通过 Issue 或 Pull Request 提出建议或贡献归档内容。

## 许可

本仓库中的内容遵循各自原始页面的许可协议。若无特殊声明，默认采用 MIT 许可证，允许自由使用、复制、修改和分发。如有疑问，请参考具体页面的注释或联系维护者。
